1
LiChat 的 search_routing 占 37.8%——assistant 侧只输出"0"或"1"（output_tokens 极短，极短占比24.1%）。是整体丢弃，还是保留 user 侧问题另作他用？整体丢弃后 LiChat 有效数据量降至约 42,000 条。

LiChat 模型来源很杂（GPT-4o 40.5% / GPT-5 17.4% / Claude-Opus 11.4% / Gemini 9.4%），不同 teacher 质量和风格差异大。是否需要只保留特定模型输出？
3
ClaudeCode 的 input_tokens 全量为 0——是上报 bug 还是设计如此？决定后续筛长文本是用字符数粗估还是必须跑一遍 tokenizer 精确计算。
